# User Preferences

## Communication Style
- Be direct and concise
- Skip unnecessary preambles
- Provide actionable information
- Use code examples over lengthy explanations

## Coding Conventions
- Prefer simple, readable code over clever solutions
- Follow existing patterns in the codebase
- Minimal dependencies when possible
- Write tests for non-trivial logic

## Tools & Environment
- Primary languages: Go, TypeScript, Python, Bash
- Version control: Git with conventional commits
- Package managers: Homebrew (macOS), npm/pnpm (Node)
- Editor: Neovim

## Workflow Preferences
- Break complex tasks into smaller, verifiable steps
- Verify changes work before moving on
- Commit logical units of work separately
